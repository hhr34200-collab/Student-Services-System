# Student Services Management System

## نظام إدارة الخدمات الطلابية

### نبذة عن المشروع

نظام إدارة الخدمات الطلابية هو نظام ويب يهدف إلى أتمتة الخدمات الأكاديمية المقدمة للطلاب داخل الجامعة، من خلال تحويل الإجراءات الورقية إلى إجراءات إلكترونية تتيح للطالب تقديم الطلبات ومتابعة حالتها، وتمكن الموظفين من مراجعتها واعتمادها وفق التسلسل الإداري المعتمد داخل الجامعة.

تم تطوير النظام باستخدام إطار العمل Laravel وفق نمط Model–View–Controller (MVC)، ويعتمد على قاعدة بيانات MySQL لإدارة البيانات، كما يوفر نظام إشعارات فورية لإبلاغ المستخدمين بحالة الطلبات أثناء انتقالها بين مراحل المراجعة المختلفة


---

# أهداف المشروع

يهدف المشروع إلى تحقيق مجموعة من الأهداف، من أهمها:

- أتمتة الخدمات الطلابية وتقليل الاعتماد على المعاملات الورقية.
- تسهيل تقديم الطلبات الأكاديمية ومتابعة حالتها إلكترونياً.
- تحسين سرعة إنجاز المعاملات داخل الجامعة.
- تنظيم سير العمل بين الطالب والموظفين والإدارة.
- توفير نظام إشعارات فورية لمتابعة الطلبات.
- توفير تقارير وإحصائيات تساعد الإدارة في متابعة أداء النظام.

# فئات المستخدمين

يتكون النظام من ثلاث فئات رئيسية من المستخدمين:

## الطالب

يستطيع الطالب إنشاء حساب، وتسجيل الدخول، وتقديم الطلبات، وإرفاق المستندات، ومتابعة حالة الطلبات، واستلام الإشعارات، وتحديث بياناته الشخصية، وطباعة الطلبات بعد اعتمادها.

## الموظف

يقوم الموظف بمراجعة الطلبات، وكتابة الإفادات، واعتماد الطلبات أو إعادتها للطالب عند الحاجة، وتحويلها إلى المرحلة التالية وفق الصلاحيات الممنوحة له.

## مدير النظام

يتولى مدير النظام إدارة بيانات النظام، بما في ذلك إدارة المستخدمين والطلاب والموظفين والكليات والأقسام والمقررات والخدمات، بالإضافة إلى متابعة التقارير والإحصائيات العامة.

# سير عمل الطلبات

تمر الطلبات داخل النظام بالمراحل الآتية:

1. تقديم الطلب من قبل الطالب.
2. مراجعة الطلب في شؤون الطلاب، ثم اتخاذ أحد الإجراءات التالية: **الموافقة وإحالته إلى المرحلة التالية، أو رفضه، أو إعادته إلى الطالب لاستكمال البيانات أو تعديلها**.
3. مراجعة الطلب من رئيس القسم، مع إمكانية **الموافقة وإحالته إلى عميد الكلية، أو رفضه، أو إعادته إلى شؤون الطلاب**.
4. مراجعة الطلب من عميد الكلية، مع إمكانية **الموافقة النهائية، أو رفض الطلب، أو إعادته إلى رئيس القسم**.
5. بعد الاعتماد النهائي تتم أرشفة الطلب وتحديث حالته داخل النظام.
6. يتم إشعار الطالب بنتيجة الطلب، سواءً **بالموافقة أو الرفض أو طلب استكمال البيانات**.

# خطوات التثبيت

بعد تنزيل المشروع من مستودع GitHub أو استنساخه إلى الجهاز، يتم تنفيذ الخطوات التالية لإعداد بيئة التشغيل:

```bash
git clone https://github.com/hhr34200-collab/Student-Services-System.git
```

الدخول إلى مجلد المشروع:

```bash
cd Student-Services-System
```

تثبيت اعتماديات PHP:

```bash
composer install
```

تثبيت اعتماديات JavaScript:

```bash
npm install
```

إنشاء ملف البيئة:

في Windows:

```bash
copy .env.example .env
```

في Linux أو macOS:

```bash
cp .env.example .env
```

إنشاء مفتاح التطبيق:

```bash
php artisan key:generate
```

# إعداد قاعدة البيانات

يعتمد المشروع على قاعدة بيانات **MySQL**.

يمكن إعداد قاعدة البيانات بإحدى الطريقتين التاليتين:

### الطريقة الأولى: استيراد ملف قاعدة البيانات

يتم إنشاء قاعدة بيانات جديدة، ثم استيراد ملف قاعدة البيانات الموجود في المسار:

```text
sql/student_serviceses.sql
```

بعد ذلك يتم تعديل بيانات الاتصال داخل ملف `.env` بما يتوافق مع بيئة التشغيل.

### الطريقة الثانية: استخدام Laravel Migrations

يمكن إنشاء الجداول الأساسية باستخدام أوامر Laravel Migrations:

```bash
php artisan migrate
```

> **ملاحظة:** تم تنفيذ بعض التعديلات على قاعدة البيانات مباشرةً باستخدام MySQL أثناء مرحلة تطوير المشروع، ولم يتم توثيق جميع هذه التعديلات ضمن ملفات الـ Migrations. لذلك قد لا ينتج عن تنفيذ أمر `php artisan migrate` قاعدة بيانات مطابقة تمامًا لقاعدة البيانات المستخدمة في المشروع، ويُنصح باستخدام ملف SQL المرفق للحصول على النسخة الكاملة.

بعد إعداد قاعدة البيانات، يتم إنشاء رابط التخزين باستخدام الأمر:

```bash
php artisan storage:link
```

# تشغيل المشروع

تشغيل خادم Laravel:

```bash
php artisan serve
```

تشغيل ملفات الواجهة الأمامية:

```bash
npm run dev
```

تشغيل خدمة Reverb للإشعارات الفورية:

```bash
php artisan reverb:start
```

تشغيل Queue Worker:

```bash
php artisan queue:work
```

بعد تشغيل جميع الخدمات السابقة، سيظهر في نافذة الطرفية (Terminal) الناتج الخاص بأمر **`php artisan serve`**، والذي يتضمن عنوان الوصول إلى التطبيق (URL). يتم نسخ الرابط الذي يظهر في ناتج هذا الأمر ثم لصقه في متصفح الويب للوصول إلى النظام

# الأجزاء غير المكتملة

لا تزال بعض الوظائف بحاجة إلى استكمال وتطوير، ومن أبرزها:

- خدمة **إعادة القيد** تم تنفيذها وربطها بالنظام، إلا أن نموذج تقديم الطلب (الاستمارة) لا يزال بحاجة إلى استكمال وإظهاره للمستخدم.
- استكمال سير عمل خدمة **التظلمات الأكاديمية** داخل لوحة الموظف، بما يشمل جميع مراحل المراجعة والاعتماد.
- إضافة بقية الخدمات الطلابية المخطط لها في النظام، حيث تم تنفيذ جزء من الخدمات فقط، بينما لا تزال بعض الخدمات الأخرى بحاجة إلى التطوير والإضافة.
